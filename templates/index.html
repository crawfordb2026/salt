Welcome to Salt

üßÇ Salt is a beginner-friendly programming language that combines the simplicity of Python with the safety of static typing. Perfect for learning programming fundamentals, Salt uses clear, readable keywords and explicit type declarations to help you understand what your code is doing.

Why Salt? Unlike complex languages that overwhelm beginners, Salt focuses on core programming concepts with intuitive syntax. Whether you're writing your first program or teaching programming concepts, Salt's straightforward approach makes coding accessible and enjoyable.

Key Features: Static type checking prevents errors, clear keywords make code self-documenting, and comprehensive error messages help you learn from mistakes. Jump into the Web IDE to start coding immediately!

Data Types

  • int - Integer numbers (e.g., 5, -10, 0)
  • double - Floating-point numbers (e.g., 3.14, -2.5)
  • string - Text strings (e.g., "hello", "world")
  • bool - Boolean values (TRUE or FALSE)

Variable Declaration & Assignment

Declaration

make int x 5
make string name "Alice"
make bool flag TRUE

Assignment

make x 10
make name "Bob"
make flag FALSE

Arrays

Declaration

make int array numbers[5]
make string array names[3]

Assignment & Access

make numbers[0] 10
make names[1] "Alice"
print numbers[0]
print names[1]

Arithmetic Operations

Salt supports standard arithmetic operations with automatic type conversion.

make int result x + 5
make double area width * height
make int difference a - b
make double ratio total / count
make int remainder 17 % 5

Print Statements

The print statement can take multiple arguments, which are automatically concatenated with spaces.

print "Hello World"
print x
print "The value is:" x
print name " is " age " years old"

Control Structures

If Statements

if x gt 5
{
    print "x is greater than 5"
}
else
{
    print "x is 5 or less"
}

Loops

Salt supports two types of loops: count-based loops and while loops.

Count-based Loops

loop 5 times
{
    print "Hello"
}

loop i from 1 to 10
{
    print "Number:" i
}

loop i from 0 to 100 by 10
{
    print i
}

While Loops

while x lt 10
{
    print x
    make x x + 1
}

Loop Control

  • skip - Skip the current iteration and continue with the next
  • end - Exit the current loop entirely
loop i from 1 to 10
{
    if i eq 5
    {
        skip
    }
    print i
}

Functions

Functions allow you to define reusable blocks of code with type checking.

make function add takes int a, int b gives int
{
    give a + b
}

make function greet takes string name gives string
{
    make string message "Hello " + name
    give message
}

Function Calls

make int result add(5, 3)
make string greeting greet("Alice")

Comparison & Logical Operators

Comparisons

  • eq - equal (==)
  • neq - not equal (!=)
  • gt - greater than (>)
  • lt - less than (<)
  • gteq - greater than or equal (>=)
  • lteq - less than or equal (<=)

Logical

  • and - logical AND
  • or - logical OR
  • not - logical NOT
x gt 5
name eq "Alice"
score lteq 100
x gt 5 and y lt 10
flag or other_flag
not finished

Unary Operations

  • - - Negation (for numbers)
make int negative -5
make int result -x

Comments

Use # for single-line comments. Comments are ignored by the interpreter.

# This is a comment
make int x 5 # Inline comment

Program Structure

  • Programs are executed line by line
  • Statements can span multiple lines (especially if statements with braces)
  • Empty lines are ignored
  • Comments (lines starting with #) are ignored
  • Functions must be defined before they can be called
  • Variables must be declared before they can be assigned new values
  • The new value must be compatible with the variable's original type

Error Handling

Salt provides comprehensive error handling with descriptive messages.

Common Errors

  • Using undefined variables
  • Type mismatches in assignments
  • Missing braces in if statements or loops
  • Invalid syntax in expressions
  • Calling undefined functions
  • Incorrect number of function arguments
  • Division by zero
  • Modulo by zero

Array-specific Errors

  • Array index out of bounds
  • Invalid array size (negative or zero)
  • Using undefined arrays
  • Type mismatch in array assignments
  • Using non-integer array indices
  • Missing array size in declaration
  • Missing brackets in array access
